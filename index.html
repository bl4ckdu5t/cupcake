This project uses 2-spaces indentation on both front-end and back-end which insinuates that the CSS, JavaScript, and Rails should be 2-spaces indented. The CSS follows the SMACSS guide, BEM, and a little AMCSS to reduce specificity spikes. No CSS frameworks are used to cut off on extra bytes for performance. No use of single line approach CSS allowed as it gets the stylesheet disorganized.

SVG should not be embedded but called as sprites where possible to keep a minified HTML file size. SVGO is used for optimizing SVG files.

A space should be left between CSS properties and values like below:

        /* Right way */
        p{
          letter-spacing: 1px;
        }
        /* Wrong */
        p{
          letter-spacing: 1px;
        }

JavaScript code should have a 'use strict' for proper error debugging

Frameworks Used

  • jQuery (Frontend)
  • AngularJS (Frontend)
  • Rails (Backend)
